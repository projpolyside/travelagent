## What is TravelAI?

TravelAI is a full-stack travel planning application that combines a **LangGraph orchestration pipeline**, a **FAISS vector RAG engine**, and **live travel data APIs** to produce detailed, budget-aware JSON itineraries for trips across India and select international destinations.

A user enters origin, destination, dates, passengers, budget, and interests. The system:

1. Resolves city names to IATA airport codes
2. Fetches live flights, hotels, weather, restaurants, and attractions in parallel
3. Retrieves relevant travel knowledge from a local FAISS vector store (RAG)
4. Generates a structured JSON itinerary via Groq (Llama 3.1 8B)
5. Returns a day-by-day plan with activities, food, transport, and costs in INR

Users can then **refine** the itinerary in plain English ("make it cheaper", "switch to train", "add more time in Old City") — the system refreshes live prices and re-generates. Every refinement is saved as a **versioned snapshot** with full rollback support.

## Features

| Feature | Description |
|---|---|
| **Full Trip Planning** | Day-by-day itinerary: morning/afternoon/evening activities, breakfast/lunch/dinner, local transport, and daily cost totals in INR |
| **Live Data Fetching** | Real-time flights (SerpAPI), hotels (SerpAPI), weather (OpenWeatherMap), and places (Google Maps), all fetched in parallel |
| **RAG Knowledge Base** | 21-document FAISS vector store covering major Indian cities with neighborhoods, pricing, timing, food, and transport details |
| **Natural Language Refinement** | Edit any aspect of the trip with a plain English instruction; system refreshes live prices before re-generating |
| **Version History & Rollback** | Every refinement is saved; roll back to any previous itinerary version |
| **Nearby Planner** | Mood-based (Relax, Adventure, Food, Romantic, Spiritual, etc.) nearby place discovery from any GPS location |
| **India Pulse** | Live travel news, festivals, and events feed powered by web search |
| **Budget Guardrails** | Hard budget constraints respected across flights, hotels, transport, and daily spend |
| **Destination Modes** | Planning style auto-detected per city: `coastal_relaxed` (Goa), `urban_heritage` (Delhi), `food_culture` (Hyderabad), `cafe_worklife` (Bangalore), etc. |
| **Version History** | Full itinerary version history per trip; roll back with one API call |
| **JWT Auth** | Secure signup/login; all trips and versions are user-scoped |
| **Groq Free Tier** | Entire LLM pipeline runs on Groq's free Llama 3.1 8B — no paid API required |

## Tech Stack

### Backend
- **FastAPI** — REST API with async support and automatic OpenAPI docs
- **LangGraph** — stateful multi-node directed graph for the planning pipeline
- **FAISS** — local vector store for RAG retrieval (no external vector DB needed)
- **SentenceTransformers** (`all-MiniLM-L6-v2`) — local 384-dim embedding model
- **Groq** (`llama-3.1-8b-instant`) — LLM for itinerary generation and refinement
- **SQLAlchemy + SQLite** — user and trip persistence (swappable to Postgres)
- **python-jose / passlib** — JWT auth and bcrypt password hashing

### Frontend
- **React 18 + Vite** — fast SPA with hot module replacement
- **React Router v6** — client-side routing across 9 pages
- **Fetch API** — direct communication with FastAPI backend

### External APIs (all optional / free-tier)
| API | Used For |
|---|---|
| SerpAPI | Live flight and hotel search |
| Serper | Fallback web search for trends and context |
| Google Maps Places | Restaurants and attractions near destination |
| OpenWeatherMap | Current weather and temperature for destination |

## Quick Start

### Prerequisites

- Python 3.11+
- Node.js 18+
- A [Groq API key](https://console.groq.com) (free, no credit card needed)

### 1. Clone and set up the backend

```bash
git clone <repo-url>
cd travel-ai-agent

python -m venv .venv

# Windows
.\.venv\Scripts\Activate.ps1
# macOS / Linux
source .venv/bin/activate

pip install -r requirements.txt
```

### 2. Set up the frontend

```bash
cd frontend
npm install
cd ..
```

### 3. Configure environment variables

Create a `.env` file in the project root:

```env
# ── Required ──────────────────────────────────────────────────────────────────
GROQ_API_KEY=gsk_...
GROQ_MODEL=llama-3.1-8b-instant
SECRET_KEY=replace_with_a_long_random_secret_string
DATABASE_URL=sqlite:///backend/travelai.db

# ── App ───────────────────────────────────────────────────────────────────────
APP_NAME=TravelAI Backend
APP_ENV=development
CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173

# ── Optional: enables live flight, hotel, and place data ──────────────────────
SERPAPI_KEY=...
SERPER_API_KEY=...
GOOGLE_MAPS_API_KEY=...
OPENWEATHER_API_KEY=...

# ── RAG configuration ─────────────────────────────────────────────────────────
RAG_EMBEDDING_MODEL=all-MiniLM-L6-v2
RAG_LOCAL_ONLY=true
RAG_CHUNK_SIZE=850
RAG_CHUNK_OVERLAP=140
RAG_TOP_K=5
RAG_SEARCH_MULTIPLIER=8
RAG_MMR_LAMBDA=0.72
RAG_MEMORY_LIMIT=350

# ── LLM ───────────────────────────────────────────────────────────────────────
PLANNER_MAX_TOKENS=4500

# ── Cache TTLs (seconds) ──────────────────────────────────────────────────────
WEB_SEARCH_CACHE_TTL=21600
WEATHER_CACHE_TTL=3600
TRANSPORT_CACHE_TTL=21600
```

Create `frontend/.env` if your backend runs on a different port:

```env
VITE_API_BASE_URL=http://127.0.0.1:8000
```

### 4. Start the backend

```bash
uvicorn backend.main:app --reload --port 8000
```

- API base: `http://localhost:8000`
- Interactive docs: `http://localhost:8000/docs`

### 5. Start the frontend

```bash
cd frontend
npm run dev
```

Open `http://localhost:5173`

## Groq Free Tier Usage

The entire LLM pipeline runs within Groq's free tier limits:

| Limit | Value |
|---|---|
| Requests / minute | 30 |
| Requests / day | 14,400 |
| Tokens / minute | 6,000 |
| Tokens / day | 500,000 |

Each trip plan uses **1 Groq call** for generation and **1 call** for refinement when requested. The itinerary prompt is engineered to stay under 2,500 output tokens. RAG context is compressed to 7 key insights before being passed to the LLM to keep the input small.
